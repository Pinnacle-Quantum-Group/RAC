The Mathematics

You won at the
hardware layer.

Claim 01
MAC is a degenerate case of RAC
Rotate vector [a, 0] by angle θ. Extract the x-component. Set θ = arctan(b/a). The result is exactly a·b. MAC is a degenerate one-dimensional projection of a two-dimensional rotation. Every multiply is a rotation you never finished.
MAC(a,b,c) = RAC([a,0], θ_b)|ₓ + c
Claim 02
RAC is strictly more expressive
A 2D rotation requires 4 MACs. RAC expresses it as 1 CORDIC sequence. Complex multiply: 4 MACs → 1 RAC. Quaternion rotation: 28 MACs → 3 RAC. The converse does not hold. RAC is strictly the more expressive primitive.
R(θ)·[x,y]ᵀ = 1 RAC vs 4 MACs
Claim 03
CORDIC uses only shift-add
Each CORDIC micro-rotation: x_{i+1} = x_i − d_i·y_i·2^{-i}. The 2^{-i} scaling is a right-shift — not a multiply. 16 iterations. Zero FPU multiplications. Converges to ±2^{-16}. Volder, 1959.
x_{i+1} = x_i − d_i · y_i · 2^{-i} ← shift
Claim 04
RAC is thermodynamically reversible
CORDIC rotations are orthogonal: det = 1, Rᵀ = R⁻¹. Information is preserved. MAC is irreversible — it destroys information at every step and Landauer's limit applies. RAC is reversible. On purpose-built silicon, that gap becomes an energy moat.
E_min(MAC) ≥ 32·k_B·T·ln(2) vs E(RAC) → 0

The SFU Insight

Every modern GPU ships with CORDIC hardware — the Special Function Units that handle sin, cos, atan. Under standard MAC workloads, those units sit 95% idle. RAC routes tensor operations through the SFUs instead of the FMA pipelines. You're not beating multiplication. You're beating the scheduler.
