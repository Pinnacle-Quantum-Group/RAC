The Mathematics

MAC is a shadow
of rotation.

Claim 01
MAC is a degenerate case of RAC
Rotate vector [a, 0] by angle θ. Extract the x-component. Set θ = arctan(b/a). The result is exactly a·b. MAC is the one-dimensional, axis-aligned projection of a two-dimensional rotation operation.
MAC(a,b,c) = RAC([a,0], θ_b)|ₓ + c
Claim 02
RAC is strictly more expressive
A 2D rotation requires 4 MACs. RAC expresses it as 1 CORDIC sequence. Complex multiplication: 4 MACs → 1 RAC. Quaternion rotation: 28 MACs → 3 RAC. The converse does not hold.
R(θ)·[x,y]ᵀ = 1 RAC vs 4 MACs
Claim 03
CORDIC uses only shift-add
Each micro-rotation: x_{i+1} = x_i − d_i·y_i·2^{-i}. The 2^{-i} scaling is a right-shift, not a multiply. 16 iterations. Zero multiplications. Converges to ±2^{-16}.
x_{i+1} = x_i − d_i · y_i · 2^{-i} ← shift
Claim 04
RAC is thermodynamically reversible
CORDIC rotations are orthogonal: det = 1, Rᵀ = R⁻¹. Information is preserved. No bits erased. MAC destroys information at every step — Landauer's limit applies. RAC approaches zero energy asymptotically.
E_min(MAC) ≥ 32·k_B·T·ln(2) vs E(RAC) → 0

The SFU Insight

Every modern GPU ships with CORDIC hardware — the Special Function Units that compute sin, cos, atan. Under MAC workloads, those SFUs are 95% idle. RAC routes tensor operations through the SFUs. The multiplier hardware is never activated.
